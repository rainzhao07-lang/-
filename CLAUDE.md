# 本命猫 H5 — 项目约定

完整需求见 `docs/本命猫H5-ClaudeCode开发任务书.md` 与 `docs/本命猫-产品设计报告-V2.md`,以下为硬红线速查:

## 红线(❌ 明确不做)
- 用户登录/注册系统(用 sessionId 定位一切)
- 管理后台 UI(管理操作走带密钥的 API + SQL)
- 小程序、i18n、暗色模式
- 任何"占卜/命理/运势/算命"类文案(含代码注释与 Prompt;表达缘分感用「契合」「匹配」)
- 在前端暴露任何密钥;LLM 调用只在服务端发生

## 工程总则
- 移动端优先(375px 基准),桌面端只需不破版
- 简单优先,禁止过度设计;不引入任务书未提及的库/服务,除非说明理由
- 引擎与内容分离:人格/题目/文案全部在 `/content` 数据文件,调优不碰组件代码
- 全站中文;视觉基调:奶油底色 + 暖橘主色,温暖治愈,拒绝廉价算命风

## 关键机制
- 计分引擎是纯函数(`lib/scoring.ts`),同样答案永远同样结果
- 报告缓存(`reports` 表):同一 session 永不重复生成、永不重复计费
- 支付走抽象接口(`lib/payment/`),MVP 为兑换码,Phase 2 换官方支付不动业务代码
- 改 `content/personas.json` 后必须重跑 `npm run font:subset`(分享卡字体是按内容子集化的)

## 本地开发的两个兜底(不影响生产行为)
- 未配 SUPABASE_URL → 内存数据库(重启即失)
- 未配 LLM_API_KEY → mock 报告流(零费用)
