# 本命猫 V1.4 一次性执行任务书(Codex 版)

> **给操作者(人类)的使用说明**:将本文件放入仓库 `docs/V1.4.md`,然后对 Codex 下达以下一条指令:
>
> ```
> 完整阅读 docs/V1.4.md,按任务组 A→B→C→D→E→F 顺序执行全部任务。附录中的文案资产必须逐字原文入库,禁止改写。全部完成后,运行 F 组回归验证并把每一项的通过/失败结果输出给我。
> ```
>
> 本任务书吸收并取代 V1.3 内容修订包。所有结论基于对当前仓库的端到端实测(本地全漏斗跑通+3份真实报告分析)。

## 第〇部分:执行者守则(最高优先级,任何任务与此冲突时以本节为准)

1. **禁止接入任何 LLM/AI 接口。** 报告保持纯算法拼装(local-rules),这是产品决策,不是技术欠账。
2. **禁止自行创作或润色任何面向用户的文案。** 所有新增文案在本文附录中给出终稿,逐字入库。若发现某处需要文案而附录未提供,留 `TODO(copy)` 注释并在完成报告中列出,不得自行编写。
3. **禁止新增 npm 依赖、禁止重构现有架构、禁止改动数据库表结构**(允许在 breeds.json / personas.json 等内容文件中新增字段)。
4. **禁止修改现有题目题面与选项文案**(仅允许按 D 组指示调整 weights 数值)。
5. **禁止改动支付/兑换模型**(一次性兑换码机制保持现状)。
6. 每完成一个任务组,运行现有测试(`npx vitest run`)确认无回归后再进入下一组。

## 第一部分:项目现状(供理解上下文,10秒读完)

产品是付费猫咪人格测试 H5:16题 → 免费结果(人格+判词+分享卡+冲突提示)→ 兑换码解锁 → 7~8道付费定制题 → 算法拼装深度报告。目前功能全部可用。本轮要解决的核心问题(实测数据):**同人格+同语气的两份报告,句子重复率73%**——闺蜜互测对比截图时会被识破为模板;外加一批实锤bug与内容资产升级。

## 任务组 A — Bug 修复(实测实锤)

**A1 双句号**:备选段拼接产生「野性优雅。。田园橘猫」。拼接前对 reason 执行 `replace(/。$/, "")`,句号由模板统一补。

**A2 标点统一**:报告引擎所有模板字符串统一为全角标点(，。：；？!),输出前对中文上下文追加 `,→，` `:→：` `;→；` 归一化。全站 UI 文案同步排查。

**A3 系统词汇泄漏**:报告中「不要把兑换报告当成接猫许可」改为「这份报告是帮你想清楚的,不是催你出发的」。全库检索,报告正文不得出现「兑换/会话/生成/session」等系统词汇。

**A4 建议行标签错位**:`housingAdvice` 把居住许可建议输出在「空间:」标签下。拆分:新增独立「居住许可:」行(仅当 consent 相关 flag 存在时输出),「空间:」只谈空间本身。

**A5 PAY_URL 兜底死胡同**:`NEXT_PUBLIC_PAY_URL` 缺失时,付费墙兜底文案「未购买请回到购买渠道获取」无任何链接。修法:①生产构建(`next build`)时若缺该变量,输出显著警告;②兜底文案改为附录 X-1 的版本(含客服出口);③README 上线清单将该变量标红为必填。

**A6 冲突信息三连轰炸降噪**:多冲突用户在一份报告中被同一串5维度清单重复数落3次。修法:
- 冲突维度优先级:`consent > medical > budget > space > shedding > allergy`
- **免费结果页提示**:最多展示优先级最高的 2 个维度
- **报告内**:「综合判断」段对 top2 深说一次;「三个风险」中的匹配风险不再罗列维度清单,改为「上面综合判断里提到的两件事,是你最该先解决的」
- 免费页提示文案模板改为附录 X-2 版本

**A7 付费后流程顺序**:确保严格为 核销成功 → 定制题 → 自动生成报告;任何中间态刷新/回退都要有明确的下一步引导,不得出现「请返回结果页输入兑换码」这类与已完成步骤矛盾的提示。

## 任务组 B — 反模板工程(头号任务:73% → ≤35%)

**B1 句级变体池**:新建 `content/reportVariants.json`,结构为 `{ 句位key: { 语气key?: string[] } }`。文案见附录 V-1(逐字入库)。选取规则:以 sessionId 哈希为种子的确定性随机——同一用户刷新永远同一版本,不同用户大概率不同。覆盖句位:开篇引入(分4语气)、"为什么是它"引入、现实适配度引言、情绪需求过渡、风险段引入、备选段引入、清单引入。

**B2 复读织入**:删除现有「其中最有参考价值的几条答案是:…→…」问答对输出(含箭头符号)。改为织入句式:按题目类别选用附录 V-2 的模板,把用户所选选项**原文**注入 `{answer}` 槽位。每份报告织入 2~3 处,位置分散(人格段1处、为什么是它1处、现实适配度1处),类别按该用户 hard_flags 中信号最强的项选取,模板变体按种子随机。

**B3 flag 堆砌句删除**:「从基础题看,你身上的线索是:作息规律、空间偏紧、…」整句删除,不做替换(信息已由 B2 与现实适配度段承载)。

**B4 备选横向对比块**(兑付付费墙承诺):用 breeds.json 现有数据渲染真对比。格式:
```
在你最在意的几个维度上,把三只放在一起看:
掉毛:{主推}偏高 · {备1}中等 · {备2}低
月花费(主粮+猫砂+护理合计):{主推}520-1090元 · {备1}…(costDetail三项区间求和)
粘人度 / 空间需求:同格式
```
维度选取:从 掉毛/月花费/空间适配/粘人度 中,取该用户 hard_flags 有信号的 3 项。对比后接一句结论,从附录 V-3 变体池按种子选取。

**B5 收尾判词**:免责声明之前插入收尾判词,每人格 3 句(附录 V-4,含新人格共10组),按种子选取。

**B6 猫名池扩容**:每个情绪需求档从 1 组扩至 3 组(附录 V-5),按种子选取。

**B7 starterKit**:breeds.json 每品种新增 `starterKit: string[]` 字段,取值按附录 V-6 的规则表**生成后直接写死入库**(不做运行时计算)。报告清单段改为:通用清单 + 「为{品种}多准备:{starterKit逗号连接}」。

**B8 相似度回归脚本**:新建 `scripts/similarity-check.mjs`:创建两个会话(答案数组见 F1)、同选温柔语气、生成两份报告、按句切分(以。!?;与换行)、输出完全相同句子的占比。该脚本纳入回归工具链。
