# 「本命猫」产品设计报告 V2.0

> V2 核心变更：① 确立产品立意「反冲动养猫」；② 付费模型从"扣押解锁"重构为"免费完整层 + 付费升维层"；③ 平台策略调整为 H5 优先、小程序后置；④ 支付方案确定为兑换码模式（零资质启动）。

---

## 1. 产品概述

**产品名称**：本命猫（对外账号名：本命猫鉴定所）

**一句话定位**：一个猫咪匹配测试——测出更契合你生活方式与情绪需求的那只猫。免费获得人格结果与分享卡，付费9.9获得完整的养猫决策报告。

**产品形态**：Phase 1 为移动端H5网页（小红书/微信/抖音引流至外链），Phase 2 数据验证后扩展为三端小程序。

---

## 2. 立意：反冲动养猫

**用最浪漫的方式，做最理性的事。**

冲动错配是弃养的主要源头。本产品的立场：花9.9元和3分钟，避免一个可能后悔几年的决定。

立意的四重商业价值：

1. **审核与合规友好**：产品定性为"宠物性格匹配/科普工具"，而非玄学测试
2. **信任即转化**：用户购买的是"决策辅助"而非"算命结果"
3. **诚意即传播**："这个测试会劝退不适合养猫的人"本身就是KOL愿意转发的信号
4. 与"领养代替购买"的价值观文案天然咬合

**皮与核分层**："本命猫"的浪漫叙事是营销的皮（站外笔记讲心动与契合）；"负责任的匹配"是产品的核（站内官方描述用"宠物性格匹配/养猫适配建议"）。站内产品文案与报告规则规避占卜类措辞。

---

## 3. 心理设计原则（继承V1）

**用户付费买的是一面美化的镜子。**

- 巴纳姆效应文案：放之四海皆准但听起来极其专属
- 奉承的艺术：缺点写成"代价"与"悲剧美"
- 人格体系中不允许存在"差"的结果
- 双重奉承：既奉承用户（天选猫奴），也拟人化奉承猫（和你一样的灵魂搭子）

---

## 4. 目标用户

18-30岁小红书/抖音活跃用户，女性为主，两类交叠：

1. **准养猫人群**：有意愿但不知如何选，缺乏研究时间 → 付费的实用理由
2. **测试爱好者**：爱玩MBTI、爱晒结果 → 免费传播的主力

用户后续价值：测试用户 = 新手铲屎官 = 宠物消费入口（Phase 3 带货/佣金空间）。

---

## 5. 付费模型：免费给体面，付费卖升维

**旧模型的病灶**：传统付费测试将用户答题应得的结果扣押勒索，产生"赎金感"，这是用户反感的唯一根源。

**新原则**：免费层完整且体面，付费层是升维服务，用户从头到尾没有一刻感觉被扣押。

### 5.1 免费层（100%用户获得）

- 铲屎官人格称号 + 判词
- 本命猫品种揭晓 + 一句"为什么是它"
- **精美竖版分享卡，免费保存与分享**（卡片自带产品入口）

> 分享卡免费是增长飞轮的燃料：玩票用户免费晒图带来流量，真准养猫用户付费买深度——用户自动分层，各取所需。

### 5.2 付费层（9.9元 · 养猫决策报告）

付费点从"看结果"移到"做决定"。报告内容：

1. 猫系人格深度剖析（含一段"被看见"的高潮文案，全报告截图率最高处）
2. 本命猫深度分析 + 另外两只备选品种及横向对比
3. 该品种真实脾气、健康雷点、月均花费、新手第一个月清单
4. 彩蛋：根据你的情绪需求生成三个猫名（超预期交付，驱动晒图）

付费墙文案写"解锁完整养猫决策报告"，不写"解锁你的测试结果"。

### 5.3 转化设计四件套

1. **价格前置透明**：落地页即标明"基础结果免费 · 深度报告9.9"，留下的用户全程无被骗感
2. **边测边懂你**：每答3-4题插入一句即时反馈（"嗯……你是个夜猫子吧？"），用户在付费墙前已亲身验证系统"真的懂我"
3. **超预期交付**：报告实际内容略多于承诺
4. **温柔兜底**：页脚"觉得不准？找客服免费重测"，消解最后一丝犹豫

---

## 6. 用户流程（全程3分钟）

1. 站外笔记/视频 → H5链接
2. 落地页（立意 + 价格透明 + 开始按钮）
3. 12题答题（含2次微反馈插入，进度条与动效）
4. 免费结果页：称号 / 判词 / 本命猫揭晓 / 免费分享卡下载 / 付费入口
5. 付费（兑换码解锁）→ 完成定制题 → 本地规则生成决策报告
6. 报告页 → 二次分享

**答题双暗线**：6题生活硬条件（作息/空间/预算/掉毛与粘人接受度/家庭结构）保证推荐可信；6题性格软偏好保证结果有人格感可晒。

---

## 7. 核心资产：人格猫矩阵

- 8-10个铲屎官人格原型，命名公式：**带暗色调的形容词 + 高贵名词**
- 每个原型绑定：主本命猫品种 + 2个备选 + 判词 + 巴纳姆式人格文案 + 微反馈语料
- 硬性规则：没有任何一个结果是"差"的
- 以独立JSON数据文件形式存在，与代码解耦，可随时调优不动工程

---

## 8. 平台策略

### Phase 1 — H5（当前阶段）

- **为什么H5**：零平台审核、随时迭代、Claude Code主场、支付资质门槛为零（见第9节）
- 小红书：笔记种草 → 评论区置顶/私信自动回复/群聊发放链接（站内不可直接外链，此为标准路径）
- 微信：链接可在内置浏览器直接打开体验
- 抖音：短视频引导（外链承接较弱，为Phase 2主升级动机）

### Phase 2 — 三端小程序（数据验证 + 个体户办妥后）

- 微信小程序：群聊原生卡片裂变 + 订阅消息触达
- 抖音小程序：视频挂载 + 达人CPS分佣分销（全新增长渠道）
- 技术：Taro/uni-app 一套代码三端，后端与H5共用
- 前置条件：个体工商户资质、iOS虚拟支付政策核实、审核措辞规范（避免占卜命理类表达）

---

## 9. 支付方案：兑换码模式（MVP）

**机制**：付费按钮 → 跳转面包多/发卡网商品页（个人可用的持牌收款平台）→ 用户支付9.9自动获得兑换码 → 返回H5输入兑换码 → 解锁报告。

- 我方代码零支付资质要求，支付行为完全发生在站外持牌平台
- 兑换码由后台预生成、一次性核销
- 工程上支付做成**可插拔模块**，Phase 2 个体户办妥后平滑替换为微信支付/支付宝官方回调，前端体验升级为一键支付

**并行行政任务（第一周启动）**：注册个体工商户（数百元）——它同时解锁：官方支付、域名备案、小程序主体，是Phase 2的总钥匙。

---

## 10. 技术方案概要

- 前端/全栈：Next.js + TypeScript + Tailwind，移动端优先
- 报告生成：本地规则 + 文案模块，根据基础答案、付费定制题、品种事实库和现实冲突生成
- 数据：Supabase（会话/兑换码/报告缓存）
- 分享卡：服务端渲染生成图片（@vercel/og）
- 部署：Vercel + 自有域名
- **已知约束**：海外部署在国内的访问稳定性存在不确定性（尤其微信内置浏览器对未备案域名偶有拦截）。对策：MVP期绑定自有域名先跑，验证成功后随个体户完成备案迁移国内节点

详细工程规格见配套文档《Claude Code 开发任务书》。

---

## 11. 增长策略（继承V1，补充）

- 标题公式：具体结果 + 情绪反应（发故事不发广告）
- 新增诚意型标题：「这个测试劝退了我养猫的念头，但我谢谢它」
- 冷启动：自产各人格结果晒图笔记铺量
- 免费分享卡 = 100%用户皆为传播节点（V1模型中仅付费用户可晒，飞轮效率提升20倍量级）

---

## 12. 风险与合规

1. 报告定位"参考建议"，内嵌免责表达
2. 全文案规避"占卜/命理/运势"类词汇（写入本地报告规则）
3. 自然植入"领养代替购买，这些性格在田园猫中同样存在"
4. 兑换码模式下客诉极少，仍设"免费重测"兜底

## 13. 路线图

**Phase 1 — H5 MVP（1-2周）**
- [ ] Claude Code 按任务书搭建工程框架（引擎）
- [ ] 人格猫矩阵 + 题库 + 本地报告文案模块（灵魂，人工打磨）
- [ ] 面包多/发卡网商品上架 + 兑换码打通
- [ ] 域名绑定 + 上线
- [ ] 冷启动笔记素材10-20篇
- [ ] （并行）个体工商户注册

**Phase 2 — 验证与小程序化**
- 漏斗数据优化（答题完成率>70%，付费转化3-8%）
- 29.9豪华版（定制视觉卡/更长报告）上线
- 微信小程序 → 抖音小程序 + 达人分佣

**Phase 3 — 商业纵深**
- 新手清单带货/佣金
- 引擎换皮：本命狗、本命植物
