# 本命猫大陆 H5 正式上线执行方案

更新时间：2026-07-07

## 推荐结论

最便宜可用方案：阿里云中国内地 ECS 99 计划，2 核 2G、3M 固定带宽、40G 云盘，99 元/年，新老用户同价，适合第一版低成本正式上线。

更适合本命猫测试期的方案：阿里云轻量应用服务器 2 核 2G。如果能抢到新人 38 元/年或 68 元/年活动价，成本最低；但活动库存和账号资格不稳定。

我不建议第一版买高配服务器。本项目当前是 Next.js H5 + 少量 API + 本地规则报告生成器，真正的成本风险不在 CPU，而在数据库、支付和备案审核。

## 购买优先级

1. 阿里云轻量应用服务器 2 核 2G活动价
   - 优点：最便宜，带宽通常比普通 ECS 更舒服，上手简单。
   - 缺点：活动价不一定随时买得到；后续扩展能力弱于 ECS。

2. 阿里云 ECS 99 计划 2 核 2G
   - 优点：99 元/年，长效、续费同价口径更清晰，可备案，正式感更强。
   - 缺点：3M 固定带宽，小红书/抖音爆一下可能会慢，后续建议加 CDN。

3. 腾讯云轻量应用服务器
   - 优点：微信生态心理上更顺，轻量产品适合小程序后端和 Web。
   - 缺点：官方常规价 2 核 2G 已到数百元/年，只有活动价才有优势。

## 当前项目生产依赖

必须配置：

- `NEXT_PUBLIC_SITE_URL=https://h5.benmingmao.cn`
- `ADMIN_SECRET=强随机字符串`
- `SUPABASE_URL=数据库地址`
- `SUPABASE_SERVICE_ROLE_KEY=服务端密钥`

注意：现在代码在没有 Supabase 时会回落到内存数据库，只适合本地演示。正式上线必须接持久化数据库，否则服务器重启后结果和付费状态会丢。

## 域名和备案

建议域名：

- 主域名：`benmingmao.cn`
- H5：`h5.benmingmao.cn`
- 后续接口可用：`api.benmingmao.cn`

大陆服务器必须做 ICP 备案。备案主体最好和未来小程序主体一致。若计划上微信、抖音、小红书小程序，建议使用个体工商户或公司主体，不建议个人主体。

备案完成后再做：

- HTTPS 证书
- 小程序业务域名配置
- CDN 加速
- 公安联网备案

## 第一次上线步骤

1. 购买服务器
   - 地域选杭州、上海、北京、广州均可。
   - 系统选 Ubuntu 22.04 LTS。
   - 开放端口：22、80、443。

2. 购买域名并实名
   - 优先买 `.cn`。
   - 域名实名主体要和备案主体一致。

3. 提交 ICP 备案
   - 网站名称建议用“本命猫鉴定所”或更合规的“本命猫养猫适配测试”。
   - 网站用途写宠物性格测试、养猫适配建议、宠物内容服务。
   - 避免写算命、命运、占卜。

4. 服务器安装环境

```bash
sudo apt update
sudo apt install -y nginx git curl
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt install -y nodejs
sudo npm i -g pm2
```

5. 上传项目并安装依赖

```bash
sudo mkdir -p /var/www/benmingmao-h5
sudo chown -R $USER:$USER /var/www/benmingmao-h5
cd /var/www/benmingmao-h5
npm ci
npm run build
```

6. 配置 `.env.local`

```bash
NEXT_PUBLIC_SITE_URL=https://h5.benmingmao.cn
ADMIN_SECRET=
SUPABASE_URL=
SUPABASE_SERVICE_ROLE_KEY=
NEXT_PUBLIC_PAY_URL=
```

7. 启动服务

```bash
sudo mkdir -p /var/log/benmingmao-h5
sudo chown -R $USER:$USER /var/log/benmingmao-h5
pm2 start deploy/ecosystem.config.cjs
pm2 save
```

8. 配置 Nginx 和 HTTPS

```bash
sudo cp deploy/nginx-benmingmao.conf.example /etc/nginx/conf.d/benmingmao.conf
sudo nginx -t
sudo systemctl reload nginx
```

HTTPS 证书可用云厂商免费证书，也可用 certbot。小程序 web-view 必须使用 HTTPS。

## 小程序内嵌路径

微信、抖音、小红书都建议只做薄壳：

```xml
<web-view src="https://h5.benmingmao.cn/?from=wechat"></web-view>
```

平台分别配置业务域名：

```text
h5.benmingmao.cn
```

如果平台要求校验文件，把平台下载的校验文件放到域名根目录可访问的位置，例如：

```text
https://h5.benmingmao.cn/平台校验文件.txt
```

## 实时更新规则

可以实时更新的是 H5 内容：

- 题库
- 结果页
- 报告文案
- 品种事实库
- 页面 UI
- 付费定制题与本地报告规则

需要重新审核的是小程序壳：

- 改 web-view 域名
- 改小程序类目
- 接入支付
- 接入登录授权
- 改原生页面结构
- 改小程序名称、头像、简介

所以生产架构要保持“小程序壳尽量薄，业务都在 H5”。

## 预计最低成本

极限省钱：

- 域名：约 1-40 元首年，取决于活动和后缀。
- 服务器：阿里云活动价 38-99 元/年。
- HTTPS：免费。
- 备案：免费。
- 数据库：初期可用 Supabase 免费额度，但大陆直连稳定性一般；正式大陆闭环建议后续迁到国内 PostgreSQL 或云数据库。
- CDN：初期可不买；有流量后再开。

更稳的商业测试预算：

- 域名：30-80 元/年。
- 服务器：99-199 元/年。
- 数据库：先用 Supabase 或 0-50 元/月国内轻量数据库。
- CDN：按量，早期通常很低。

## 我的执行建议

第一步先买阿里云 ECS 99 计划或阿里云轻量 2 核 2G 活动机。

如果你能抢到 38/68 元轻量机，就选轻量机；如果抢不到，选 99 元 ECS。不要为了第一版测试直接买 4 核 8G。

第二步域名买在同一个云厂商，备案最省心。

第三步先上线 H5，微信小程序壳随后接入。抖音和小红书等 H5 稳定后再做。
