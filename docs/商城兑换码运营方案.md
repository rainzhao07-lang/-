# 商城兑换码运营方案

## 当前路线

用户在小红书商城或其他商城购买“本命猫深度报告兑换码”，收到兑换码和测试网址后，进入 H5 完成测试并解锁报告。

这条路线的重点是：交易先放在商城，H5 负责测试、核销和报告生成。

## 用户路径

1. 用户在商城下单。
2. 商城自动发货兑换码和网址。
3. 用户打开网址，粘贴兑换码。
4. 用户完成 16 道题。
5. 结果页自动带入兑换码。
6. 用户点击“解锁报告”。
7. 系统核销兑换码并生成深度报告。

## 商品标题建议

优先使用：

```text
本命猫养猫适配测试｜深度报告兑换码
```

可选标题：

```text
本命猫鉴定所｜养猫决策报告兑换码
本命猫测试｜猫咪性格适配深度报告
```

避免使用：

```text
算命
占卜
命运预测
心理诊断
医疗建议
保证适合
```

```text
2. 输入兑换码
3. 完成 16 道题
4. 在结果页解锁专属养猫决策报告

报告内容包括：
- 你的猫系人格
- 适配猫品种建议

本测试与报告仅作养猫决策参考，不构成专业医疗、心理或法律意见。每只猫都是独立个体，请结合现实情况理性判断。
```

## 退款和售后建议

商品页建议写清：

```text
本商品为数字内容兑换码。兑换码未使用前，如因重复购买等原因可联系客服处理；兑换码一经核销并生成报告，原则上不支持无理由退款。
```

不要写得过硬。平台更看重售后体验，建议保留人工兜底。

## 后台生成码

接口：

```text
POST /api/admin/codes
```

鉴权：

```text
x-admin-secret: 你的 ADMIN_SECRET
```

JSON 返回：

```json
{
  "count": 10,
  "channel": "xiaohongshu",
  "batch": "2026-07-07"
}
```

CSV 导出：

```json
{
  "count": 10,
  "format": "csv",
  "channel": "xiaohongshu",
  "batch": "2026-07-07"
}
```

CSV 字段：

```text
code, redeem_url, channel, batch, delivery_text
```

`delivery_text` 可以直接复制到商城自动发货内容里。

## 权限原则

- 普通用户只能访问测试、结果、兑换、报告页面。
- 批量生成兑换码只通过 `ADMIN_SECRET` 鉴权。
- `ADMIN_SECRET` 只给你和伙伴，不放进前端，不发给外包或测试用户。
- 正式上线前必须配置持久化数据库和模型密钥。
